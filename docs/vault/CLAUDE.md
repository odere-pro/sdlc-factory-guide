# Wiki Schema — SDLC Factory Guide

This vault powers the wiki for the **sdlc-factory-guide** project: an opinionated guide
to running software-development lifecycle workflows with AI agents, covering planning,
context engineering, cost control, code review, verification, and shipping.

## Purpose

The wiki extracts structured, citeable knowledge from the project's documentation so
that engineers can query it with natural language rather than reading every guide in
sequence. Pages are auto-generated by the ingest pipeline and should not be edited
by hand — edit the source documents in `docs/` instead.

## Directory layout

| Path | Role |
|---|---|
| `raw/` | Immutable source files. Drop files here to queue them for ingest. |
| `raw/wired/` | Snapshots from wired git sources (created by wire-source.sh). |
| `wiki/` | Generated wiki pages — owned by the ingest pipeline. |
| `wiki/index.md` | Auto-generated topic index (regenerated after every ingest). |
| `wiki/log.md` | Running ingest log — one entry per run. |
| `CLAUDE.md` | This file. Schema authority for all vault writes. |

## Page classes

The ingest pipeline produces pages in nine typed classes:

1. **concept** — definitions and mental models
2. **process** — step-by-step workflows
3. **decision** — ADRs and design choices
4. **reference** — commands, APIs, configuration tables
5. **example** — concrete code or workflow examples
6. **overview** — high-level summaries of a topic area
7. **comparison** — side-by-side trade-off tables
8. **faq** — question-and-answer collections
9. **glossary** — term definitions

## Frontmatter requirements (all wiki pages)

```yaml
---
title: <page title>
page_class: <one of the nine classes above>
source: "[[raw/filename.md]]"
tags: [<topic tags>]
---
```

## Wikilink convention

Internal cross-references use Obsidian-style wikilinks: `[[Page Title]]`.
Source citations appear as `[[raw/filename.md]]`.
