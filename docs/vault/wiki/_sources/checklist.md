## Summary

A flat, actionable task list spanning all eight parts of the guide. Each section mirrors one part of the main series and provides checkbox items an engineer can work through to implement that part of the workflow. Referenced from the guide as the companion implementation resource.

## Key Claims

Eight sections, one per guide part. Part 1 (Rule File): create CLAUDE.md with stack/conventions/hard rules/workflow. Part 2 (Context): separate static from dynamic, build skills. Part 3 (Verification): write tests first, build evals with rubrics. Part 4 (Run the Work): pick mode per task (conductor vs orchestrator), sandbox execution. Part 5 (Review): agent first-pass, hook enforcement, observability. Part 6 (Cost): measure TCO, route models, use dynamic context. Part 7 (Production): add persistent memory/scoped permissions/CI evals for products. Part 8 (Team): version the harness, gate on evals, hire for judgment. Covers: Implementation Checklist, Rule File, Context Engineering, Verification, Running the Work, Controlling Cost, Production Agents, Team Standard.
